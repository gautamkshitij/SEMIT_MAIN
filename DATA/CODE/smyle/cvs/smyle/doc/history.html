SMYLE: History

Changes from 0.8.2 to 0.8.3

  • Much more scalable index architecture
  • Support for field type sequence<byte>

Changes from 0.8.1 to 0.8.2

  • JDK 1.4 compatibility

Changes from 0.8 to 0.8.1

  • Support for read-only media (e.g. CDROM)
  • Swing-based database browser
  • Support for recursive type definitions
  • Improved schema evolution
  • Performance improvements

Changes from 0.2.5 to 0.8

  • Automagic multidimensional indexing
  • Self-contained code generator that directly outputs .class files
  • Optimized for plain Java (while still compatible with GJ)
  • Vastly improved performance on big tables
  • Can efficiently handle 100,000+ rows
  • More powerful filters
  • Store definition can now be done in IDL
  • Lots of small improvements

Changes from 0.2 to 0.2.5

  • New, compact store format
  • Schema evolution
  • Table implements Collection API
  • New notion of mutable snapshots for write access
  • Write access is fully synchronized (no more ConcurrentWriteExceptions!)
  • Thread safety
  • API changes and improvements

Changes from 0.1 to 0.2

  • Stores are now garbage collected
  • Much richer API
  • Big improvements in safety and concurrency
