SMYLE: Users

Who is using Smyle?

Smyle is successfully being used in production applications. Here are a few examples:

A Community System (Chat/Forum/Mail/Guestbook)

This is a web-based community system developed by the author of Smyle. Currently it is also the main application that drives the development of Smyle itself.

The sources are not available yet, but I plan to include some of them in the Smyle distribution as a sample application.

As the community server is upgraded to the latest development version of Smyle very often, you might expect that it's quite unstable. Surprisingly, this is not the case. The record so far:

  • Loss of data - not once.
  • Availability - very high. The only db-related incident was a corrupted index structure caused by a pre-0.3 bug that prevented some of the users from logging in.
  • Performance - good. Typical page latency <1s
Some more facts:
  • URL: www.fluesterstern.de (site is in German)
  • Technologies employed: Smyle, WebMacro, Tomcat
  • Language used: GJ
  • Tables: ~15
  • Rows per table: ~100-20,000
  • Write transactions so far: >300,000

JImageDB

A system of applications that can package a categorized, keyword-searchable set of image files onto a CD.
  • Author: James Damour
  • URL: http://jimagedb.sourceforge.net
  • Smyle's role: provide the persistent data store for the categories and keywords, and their relation to the image files
  • Rows per table: up to a few thousand
  • Technologies employed: Smyle, Servlets, Swing
Author: Stefan Reich (doc@drjava.de)
Smyle Homepage: www.drjava.de/smyle
